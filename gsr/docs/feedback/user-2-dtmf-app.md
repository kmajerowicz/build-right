# User 2 Feedback — DTMF Demo App (April 2026)

## User Profile

- Developer, techniczny
- Buduje dla zabawy (demo dla córki)
- Sam zaczął pisać appkę przed użyciem GSR — wiedział jakie podejście chce zastosować
- Podał konkretny stack w pierwszym prompcie (Python, PySide, numpy, sounddevice)

## GSR Version Tested

0.1.x (pre-feedback release)

## First Prompt

> "I want to create an app to demonstrate DTMF codes for my kid. It should be one window with standard numeric keyboard in a grid. I would like to have labels first row and header describing correct frequencies. After clicking each button on a keyboard, the sound should play. It should be a Python app using PySide, numpy, sounddevice libraries. Sounds should be autogenerated on the fly."

## Raw Feedback (original, Polish)

Mam bardzo pozytywne wrażenia. Etap planowania był super. Następnym razem przetestuję jak to wygląda jak nie dam mu żadnych podpowiedzi co do technologii.

Bardzo dobrze napisał PRD i dokumentację przy praktycznie moim żadnym wkładzie własnym.

Testy human in the loop są bardzo spoko.

Jedyne co to dla mnie w tym małym projekciku totalnie niepotrzebne były sugestie co do v2 - czułem jakby to był taki feature creep, który mnie odciąga od core mojej aplikacji.
